Informing the User

The "Blank Slate" State

Instead of displaying empty tables, lists or areas in the UI, the console should provide helpful text and actions.

For example, instead of displaying an empty table with a search bar when there are no deployed applications, encourage the user to deploy an app and link them to the appstore, to the client, and to the online docs about deploy apps.

Destructive Actions

Users should always be warned and presented with confirmation before a destructive action is performed.

Destructive actions should always be marked by red and clearly indicate that they will delete the entity.

Note: This should not be confused with non-destructive removals. (eg. removing a user from an organization doesn't destroy the user) In this case, the action should be in orange (our warning colour) instead of red.

Who is That?

It is important to show users who performed actions, and give them context in the timeline and when displaying lists of users.

It is insufficient to just show a Gravatar. At minimum, Gravatars must have tooltips that display the user's email address and/or username. Preferably, Gravatars would be accompanied by display of the user's email address, user and display name (if set).

Note: We may want to consider implementing User Cards.

    Navigation

    Breadcrumbs

    Breadcrumbs should be used when the user is multiple-levels deep (ie: viewing an app, which is part of a space, which is part of an org). They should always be placed below the view's page header. Each piece of the breadcrumb should be linked to it's corresponding view.

    env-123ab

    my_org / my_space 
    <div class="page-header">
    <h1>
        <span class="glyphicon glyphicon-th-large"></span>
        <span class="app-name"> env-123ab</span>
    </h1>
    <span class="text-small">
        <a href="">my_org</a> /
        <a href="">my_space</a>
    </span>
</div>

    In-View Navigation

    When there is too much content to display all at once, vertical navigation tabs should be used. This gives us room to expand the views in the future to display even more information.

    Note: If there is information in a view that is contextually relevent to information presented in multiple tab panels, it should be displayed outside of the tabbed navigation

    Components

    Buttons

    Buttons that exist outside of tables and lists, should have both an icon and a text description

    <button class="btn btn-primary"><span class="glyphicon glyphicon-plus"> Add User</span></button>

    Buttons in table rows or lists should just be an icon, with text in a tool-tip.

    Bob Bobson
    Jane Janeville

    Buttons should be colour coded to their actions. Safe actions should be blue, non-destructive removals should be orange, and destructive actions should be red.

    <button class="btn btn-primary"><span class="glyphicon glyphicon-plus"></span> Safe action</button>
<button class="btn btn-warning"><span class="glyphicon glyphicon-minus"></span> Non-destructive removal action</button>
<button class="btn btn-danger"><span class="glyphicon glyphicon-trash"></span> Destructive action</button>

    IconClass namesUse
    icomoon-pencil Editing values
    icomoon-plus Creation of entities/addition of records
    icomoon-remove Destructive removal (ie: deleting a user)
    icomoon-minus Non-destructive removal (ie: removing admin status from a user)

    Sliders

    Sliders should be used when settings require a range of values. The values should be displayed to the user above the slider, and if the slider saves automatically, there should be a notice that appears to the right of the values showing the result of the save.

    CPU Autoscaling

    Saving... Saved! Save failed! 

    <div class="panel panel-default">
    <div class="panel-heading">CPU Autoscaling</div>
    <div class="panel-body">
        <div class="col-md-12 col-lg-6 slider-block" id="cpu-threshold">
            <p>
                <small class="label label-default slider-saving" id="cpu-threshold-saving">Saving...
                </small>
                <small class="label label-success slider-saved" id="cpu-threshold-saved">Saved!
                </small>
                <small class="label label-danger slider-failed" id="cpu-threshold-failed">Save failed!
                </small>
                <label for="cpu-threshold-label">
                    CPU Threshold:</label>
                <input type="text" id="cpu-threshold-label" class="slider-label">
            </p>
            <div id="slider-cpu-threshold" class="slider-container"></div>
        </div>
    </div>
</div>

<script>
    sliderSlid = function (slider_el, unit_indicator, ui) {
        $(".slider-label", slider_el).val(ui.values[ 0 ] + unit_indicator + " - " + ui.values[ 1 ] + unit_indicator);
    },

    $("#slider-cpu-threshold").slider({
        range: true,
        min: 0,
        max: 100,
        values: [0, 50],
        slide: function (event, ui) {
            sliderSlid('#cpu-threshold', "%", ui);
        }
    });
    $("#cpu-threshold .slider-label").val($("#cpu-threshold .slider-container").slider("values", 0) +
        "% - " + $("#cpu-threshold .slider-container").slider("values", 1) + "%");
</script>
